Se muestran las reglas básicas para diseñar un JavaBean:

  • Se define como clase pública.
  • Debe tener un constructor vacío por defecto, aunque puede tener más constructores.
  • Se debe implementar el interface Serializable para poder implementar la persistencia.
  • Se deben seguir las convenciones de nombres establecidas, como por ejemplo los metodos get...() y set...() se usan para obtener y establecer propiedades de un Bean.
